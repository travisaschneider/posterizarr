# Security Policy

Thank you for helping keep Posterizarr secure. 

## Supported Versions

Currently, only the latest release of Posterizarr is actively supported with security updates. We strongly recommend keeping your installation up to date.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Latest Dev | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

How you should report a vulnerability depends on its severity. 

### Critical or High Severity
If you discover a critical or high-severity security issue, **please do not open a public GitHub issue.** Instead, report it to me privately so it can be patched before the details become public.

* **Discord:** Send me a direct message (PM) at `fs.corrupt`.

I will try to respond to critical reports as quickly as possible to validate the issue and coordinate a fix.

### Non-Critical or Low Severity
For minor security issues, non-critical bugs, or general security-related feature requests, you can use the standard public tracking process:

* **GitHub:** Open a Security [Issue](https://github.com/fscorrupt/posterizarr/issues/new?template=-security--non-critical-security-report.md) in the Posterizarr repository.

---
*By contributing a report, you are helping improve the safety and reliability of Posterizarr for the whole community.*
